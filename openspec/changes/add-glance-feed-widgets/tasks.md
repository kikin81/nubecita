## 3. Image-prefetch seam in the worker (D-C4) — modifies widget-feed-refresh

- [x] 3.1 Added `interface WidgetImagePrefetcher { suspend fun prefetch(feedKey: FeedKey) }` + `NoOpWidgetImagePrefetcher` to `:core:widget-sync` (mirrors `WidgetUpdater`; no `androidx.glance`/image-loader dep). Bound at the `:app` flavor level + the androidTest test module (renamed the group-2 `WidgetUpdaterModule`/`TestWidgetUpdaterModule` → `WidgetSeamBindingsModule`/`TestWidgetSeamBindingsModule`, each now binding both seams) — same flavor-relocation pattern as the updater.
- [x] 3.2 `WidgetRefreshRunner` injects `WidgetImagePrefetcher` and calls `prefetch(feedKey)` after a feed's `trimToCap` succeeds, in its **own** nested try/catch (rethrow `CancellationException`; a prefetch throw is logged and fails only that feed's images — the refresh already succeeded, so it never flips the outcome, skips the updater, or aborts other feeds). `WidgetRefreshRunnerTest` extended (8 tests): prefetch invoked once per succeeded feed (order Following→Discover), not at all when every feed fails, only the succeeded feed on partial failure, and a throwing prefetcher leaves the worker `SUCCESS` with the other feed still refreshed/trimmed/prefetched + updater run once.

## 4. Thumbnail store + prefetch implementation (D-C5)

- [x] 4.1 Implemented `WidgetThumbnailStore`: dedicated cache dir `cacheDir/widget_thumbs/<sanitized account>/<sha256(postId)>.jpg`, `thumbnailFile`/`hasThumbnail`, `evict(keep)`, `clearAccount(did)`. Pure file I/O (root `File` injected — `<cacheDir>/widget_thumbs` via the `@Inject` ctor), so no `Bitmap` dep. JVM unit test (5, temp dir): deterministic key→file mapping under the account dir, `hasThumbnail` round-trip, evict prune (keeps head, drops the rest), empty-keep clears, `clearAccount` scoped to one account.
- [x] 4.2 Implemented the real prefetcher `GlanceWidgetImagePrefetcher` + the `ThumbnailDecoder` seam (`CoilThumbnailDecoder` — the only Android/Coil-bound step: `ImageLoader.execute` → `toBitmap` → JPEG to file, `allowHardware(false)`, fixed 600px box; bound module-internal via `WidgetImageModule`). The per-embed decision is the pure `widgetThumbnailUrl(embed)` (Images first thumb / Video poster / RecordWithMedia→media; External/Gif/Record/Empty/Unsupported + content-warned media → null). JVM tests: `EmbedThumbnailsTest` (7, per-variant) + `GlanceWidgetImagePrefetcherTest` (3, fake decoder + mockk repo + real store) — one decode per media post, text-only skipped, already-cached skipped.
- [x] 4.3 Eviction lives in the store (`evict(accountDid, keepPostIds)` = delete files whose post isn't in the head set) and is called by the prefetcher with the current `head(n)` ids each run; `clearAccount` available for logout wiring. JVM-tested via the store evict cases + the prefetcher's "evicts posts no longer in the head" test. (Coil decode + file write + the budget check are deferred to the instrumented pass, task 9.1.)

- [x] 6.1 Implemented the post-tap deep link: pure `widgetPostDeepLink(postUri)` → `nubecita://profile/<authority>/post/<rkey>` (the scheme/host `MainActivity` registers; `:feature:profile:impl`'s matcher routes it to `PostDetailRoute`). `FeedWidget.provideGlance` builds a per-row `ACTION_VIEW` intent scoped to our package; `PostRow` wraps it via `GlanceModifier.clickable(actionStartActivity(intent))` (no `clickable{}`-lambda launch — Android-12 trampoline-safe). `WidgetRow.deepLinkIntent` carries it (null in unit tests, which can't build a real `Intent` on the JVM). JVM test `WidgetDeepLinksTest` (2: did- + handle-authority). The "row carries the `actionStartActivity` Action" assertion is deferred to the instrumented pass (task 9.1) — `actionStartActivity(Intent)` needs a real `Intent`, which throws "Stub!" under JVM unit tests.
- [x] 6.2 Added a public `WidgetRefreshLauncher` seam to `:core:widget-sync` (the scheduler stays `internal`; `DefaultWidgetRefreshLauncher` delegates to `WidgetWorkScheduler.refreshNow()` → unique one-time `KEEP` work), exposed via the `WidgetEntryPoint`. **Manual refresh**: a header refresh icon wired with `clickable(actionRunCallback<RefreshWidgetAction>())`; the `ActionCallback` resolves the launcher via the EntryPoint and enqueues. **Widget-add / not-yet-populated**: `provideGlance` enqueues a refresh when the head is empty (self-healing, KEEP-deduped) — chosen over a receiver `onUpdate` override (cleaner; no goAsync/PendingResult juggling). Widget render path issues no network. Glance test asserts the refresh affordance composes in every state (5 `FeedWidgetContentTest`); the run-callback/enqueue correctness is instrumented (9.1). 31 module tests.
