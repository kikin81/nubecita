## 2. PR 2 — Report dialog screen + ViewModel + `@MainShell` provider

- [x] 2.1 Define `ReportDialogContract.kt` in `:feature:moderation:impl`: `ReportDialogState` (per the design's Decision 3 shape, including derived `canSubmit: Boolean`), `ReportDialogEvent` (`OnCategorySelected`, `OnReasonSelected`, `OnDetailsChanged`, `OnSubmitClicked`, `OnBackPressed`, `OnCancelClicked`), `ReportDialogEffect` (`RequestDismiss`, `Submit` if needed for tests), `ReportDialogStep` sealed sum (`Subject | Category | SubReason | Details`), `SubmissionStatus` sealed sum (`Idle | Submitting | Success(sentAt: Instant) | Failed(message: String)`), `SubjectPreview` data class
- [x] 2.2 Implement `ReportDialogViewModel` extending `MviViewModel<ReportDialogState, ReportDialogEvent, ReportDialogEffect>`: inject `ModerationRepository` + `SavedStateHandle` (the SavedStateHandle delivers the `Report` NavKey via the Nav3 entry installer's serialized args); init block parses the NavKey's `subject` and resolves the `SubjectPreview` via the repository (post snippet for Post; profile handle + display name for Account)
- [x] 2.3 Implement step transitions in the reducer: `OnCategorySelected(category)` → `step = Category` if not already, or `step = SubReason` if `category.reasons.size == 1` (Spam fast-path) else `step = SubReason`; `OnReasonSelected(token)` → set `selectedReason`, set `detailsRequired = token in OTHER_REPORT_REASONS`, `step = Details`; `OnDetailsChanged(text)` → truncate at 300 graphemes via `:core:common`'s `BreakIterator` helper, set `details = truncated`; `OnBackPressed` → step backward (Details → SubReason → Category → Subject) and clear the field set during the abandoned step
- [x] 2.4 Implement `canSubmit` derivation in the reducer per the spec: `selectedReason != null && (!detailsRequired || details.graphemeCount in 1..300) && submission !is Submitting`
- [x] 2.5 Implement `OnSubmitClicked` handler: transition `submission` to `Submitting`; call `repository.reportPost(...)` or `reportAccount(...)` per `subject` variant; on success → `SubmissionStatus.Success(sentAt = now)` and schedule a `viewModelScope.launch { delay(2500); sendEffect(RequestDismiss) }`; on failure → `SubmissionStatus.Failed(message = exception.localizedMessage ?: fallbackString)`
- [x] 2.6 Implement `ReportDialogScreen.kt` (stateful) and `ReportDialogContent.kt` (stateless) split, following the `FeedScreen` / `FeedScreenContent` pattern. The stateful wrapper consumes `hiltViewModel<ReportDialogViewModel>()` and collects effects via a single `LaunchedEffect`
- [x] 2.7 Implement the Subject step Composable: header card showing post snippet + author (for `ReportSubject.Post`) or profile handle + display name (for `ReportSubject.Account`); render a skeleton state while `subjectPreview == null`; Continue CTA dispatches `OnCategorySelected` → triggers transition to step Category
- [x] 2.8 Implement the Category step Composable: 9 cards in a `LazyVerticalGrid(GridCells.Fixed(2))` or `Column`; each card uses `NubecitaIcon` + label; tap dispatches `OnCategorySelected(category)`
- [x] 2.9 Implement the SubReason step Composable: `LazyColumn` of reason rows (icon + label) keyed by token string; tap dispatches `OnReasonSelected(token)`; Spam fast-paths through SubReason without rendering a list (handled in reducer)
- [x] 2.10 Implement the Details step Composable: `OutlinedTextField` with `state.details`, `onValueChange = { dispatch(OnDetailsChanged(it)) }`, live `<n>/300` grapheme counter, `isError = detailsRequired && state.details.isBlank()`. Inline error banner above the Submit CTA when `submission is Failed`. Submit CTA `enabled = state.canSubmit`. Auto-focus the field on step entry
- [x] 2.11 Implement the Success card Composable: replaces the form content when `submission is Success`. Renders the success copy + a single Dismiss tap region (whole sheet). VM's 2.5s delay auto-emits `RequestDismiss`
- [x] 2.12 Wire `BackHandler { dispatch(OnBackPressed) }` with `enabled = state.step != ReportDialogStep.Subject`; the dialog's `ModalBottomSheet.onDismissRequest` always fires `OnCancelClicked` which emits `RequestDismiss`

The provider Composable wraps `ReportDialogScreen` in `ModalBottomSheet(onDismissRequest = { mainShellNavState.removeLast() })`. No `listPane{}` / `detailPane{}` metadata
- [x] 2.14 String resources in `:feature:moderation:impl/src/main/res/values/strings.xml`: every label visible in the dialog (category names, sub-reason labels, step CTAs, success copy, fallback error copy)
