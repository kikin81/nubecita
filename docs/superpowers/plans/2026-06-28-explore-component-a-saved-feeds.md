# Explore Component A — Room-cached Saved Feeds (nubecita-4qpv) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make the user's saved/pinned feeds Room-backed (offline-first read, instant render) and add **non-destructive** pin/unpin with write-through. Foundation for Component C's inline Pin and faster pinned-feed rendering everywhere.

**Architecture:** A new `SavedFeedEntity`/DAO in `:core:database` (display-ready rows). `:core:feeds`'s `PinnedFeedsRepository` becomes offline-first: `observePinnedFeeds(): Flow` (emits from Room) + `refresh()` (network → cache-safe write-through), plus `pinFeed`/`unpinFeed` (a `putPreferences` read-modify-write of `SavedFeedsPrefV2` under a mutex, mirroring `:core:moderation`). Full design: `docs/superpowers/specs/2026-06-28-explore-page-design.md`.

**Tech Stack:** Room (`:core:database`, room convention plugin, schema export), atproto-kotlin (`ActorService.getPreferences/putPreferences`, `FeedService.getFeedGenerators`, `SavedFeed`/`SavedFeedsPrefV2`), Hilt, JUnit5 + MockK + Turbine (JVM repo tests), AndroidJUnit4 + in-memory Room (instrumented DAO/migration tests).

## Global Constraints

- **Non-destructive unpin:** AT Proto distinguishes *saved* from *pinned*. `unpinFeed` sets the existing `SavedFeed.pinned = false`; it MUST NOT drop the item (would delete a feed saved on another client). `pinFeed` upserts the item with `pinned = true`.
- **Cache-safe refresh:** `refresh()` must never wipe the last-good Room cache when `getFeedGenerators` fails (full or partial) — abort the write or merge (retain old metadata for unresolved URIs), and write via **diff-upsert (upsert + delete-not-in), not a table-nuke**, so already-rendered pinned icons don't flicker.
- **Preserve foreign preferences:** the `putPreferences` write must round-trip every OTHER preference entry untouched (mirror `mergeModerationPrefs`, remapping foreign GET-union members via `asPutPreference()`).
- **`:core:database` DAO + migration tests are `androidTest` (instrumented)** — they need a connected device / the `run-instrumented` PR label, not `testDebugUnitTest`. `:core:feeds` repo tests are JVM unit tests (mock the DAO + `FeedsDataSource`).
- No Co-Authored-By/Claude-Session commit footer. Conventional Commits, lowercase subject. Never `--no-verify`.

**Interfaces — Produces:**
```kotlin
@Entity(tableName = "saved_feeds")
data class SavedFeedEntity(
    @PrimaryKey val uri: String,
    @ColumnInfo(name = "display_name") val displayName: String,
    @ColumnInfo(name = "creator_handle") val creatorHandle: String?,
    @ColumnInfo(name = "avatar_url") val avatarUrl: String?,
    val pinned: Boolean,
    val position: Int,
)
@Dao interface SavedFeedDao {
    @Query("SELECT * FROM saved_feeds ORDER BY position ASC")
    fun observeSavedFeeds(): Flow<List<SavedFeedEntity>>
    @Upsert suspend fun upsert(feeds: List<SavedFeedEntity>)
    @Query("UPDATE saved_feeds SET pinned = :pinned WHERE uri = :uri") suspend fun setPinned(uri: String, pinned: Boolean)
    @Query("DELETE FROM saved_feeds WHERE uri NOT IN (:keepUris)") suspend fun deleteUrisNotIn(keepUris: List<String>)
}
```
